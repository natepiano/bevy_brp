List curated BRP records published by the running application's developer.

The public structured result puts instructions in result.usage and records in result.tools. Each
record includes its exact backing method and optional raw parameter/result JSON Schemas. Follow the
usage, then pass a selected method and matching raw params to brp_execute. Records are not native
MCP tools; rpc_discover instead exhaustively lists transport method names.

Each request validates every record against the live instant methods and returns either the full
list or an error if any method is missing or watching. A missing endpoint means the app needs
BrpExtrasPlugin; an installed endpoint may return an empty list. Direct HTTP JSON-RPC is a
developer-only alternative.
